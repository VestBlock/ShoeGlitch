# operator-email-ops

Use this skill when work touches operator application emails, onboarding emails, or operator operations messaging.

## Objective
Turn operator email into an operational funnel:
- acknowledge intent
- notify admin
- move qualified applicants forward
- keep messaging realistic

## Rules
- Keep operator earnings and territory language grounded. Do not fabricate upside.
- Admin alerts should include enough context to review quickly:
  - city
  - tier / role focus
  - contact info
  - experience
  - why they applied
- Applicant emails should clearly tell them what happens next.
- Approval and rejection emails must map to real application state, not aspirational future states.

## Core flows
- application received
- admin alert
- approved: next step / onboarding / payment if applicable
- rejected or not moving forward
- city waitlist / launch-interest follow-up

## Anti-patterns
- operator emails that read like generic hiring blasts
- copy that conflicts with the actual application or payment flow
- silent admin dependency with no alert path

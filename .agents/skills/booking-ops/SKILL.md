# booking-ops

Use this skill when working on service packages, booking flows, pickup/drop-off logic, steam-cleaning messaging, or operator package alignment.

## Objective
Keep ShoeGlitch’s service promises, package copy, and operational workflow aligned.

## Rules
- Do not change booking logic casually.
- Keep package messaging consistent across homepage, services, booking, operator pages, and SEO pages.
- Steam-assisted cleaning is part of every package except the lowest entry tier.
- Highest operator packages should clearly include the steam cleaner brush setup when operator kit content is involved.

## Focus areas
- package comparison clarity
- pickup/drop-off promise accuracy
- booking friction reduction
- operator package expectations

## Anti-patterns
- package copy drifting across pages
- promising operational features not supported in the flow
- changing user-facing package language without checking operator implications

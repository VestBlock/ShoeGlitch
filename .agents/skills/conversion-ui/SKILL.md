# Purpose
Build money pages, not pretty dead ends.

# Use this skill when
- Designing landing pages, feed cards, detail pages, quote forms, booking-entry points, sticky CTAs, upsells, or monetization surfaces.

# UX priorities
- attention
- clarity
- value stacking
- friction reduction
- speed

# Rules
- Users should know what to do in the first screen.
- One primary CTA per viewport section.
- Secondary actions support the primary CTA instead of competing with it.
- Put proof and trust near the action, not buried below.
- Design mobile first. If mobile is unclear, the design is wrong.

# Good CTA patterns
- `Book cleaning`
- `Restore this pair`
- `Check your ZIP`
- `Watch market`
- `Join waitlist`
- `Get a quote`

# Layout rules
- Strong visual hierarchy on cards:
  - item identity
  - opportunity signal
  - proof
  - CTA
- Sticky actions are useful when the page is long or data-dense.
- Images are important, but performance and scanability matter more than decoration.

# Trust elements
- service area coverage
- tracking or transparency
- before/after logic
- clear pricing starting points
- social proof when available

# Anti-patterns
- Do not bury the action below the fold without a reason.
- Do not use vague CTA labels.
- Do not create visually rich blocks that do not move the user toward booking or lead capture.
- Do not let glassmorphism or effects reduce readability.

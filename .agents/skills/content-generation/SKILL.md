# Purpose
Build content systems that answer, qualify, and convert.

# Use this skill when
- Writing or templating city pages, service pages, guides, FAQs, summaries, or comparison content.

# Content rules
- Start with the direct answer.
- Use real service language from ShoeGlitch.
- Keep paragraphs scannable.
- End sections by moving readers toward the next action.

# Required section types
- quick answer
- long answer
- what to expect
- local proof or use case
- FAQ
- CTA

# Tone
- direct
- confident
- practical
- premium but not inflated

# Anti-patterns
- Do not generate generic “best in town” fluff.
- Do not overstate what the service can fix.
- Do not publish long-form text with no clear booking path.

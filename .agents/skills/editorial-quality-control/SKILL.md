# editorial-quality-control

Use this skill when reviewing public-facing copy for mock text, internal language, awkward placeholders, or thin SEO content.

## Objective
Catch public copy that sounds like build notes instead of live product language.

## What to remove or rewrite
- internal phrases like `scaffolded`, `AI-readable`, `conversion path`, `use realistic language`
- implementation notes exposed to users
- thin keyword-swapped filler
- copy that sounds like a prompt rather than a page

## Rules
- Public copy should sound helpful, direct, and credible.
- Internal accuracy is more important than hype.
- Preserve the route’s search intent while cleaning the language.

## Anti-patterns
- leaving debug or author notes in cards
- using meta commentary as visible content
- repeating the same headline or summary block across route families
